# 抄送（Carbon Copy）

抄送功能会在发送到 Telegram 的同时，将 Telegram SMS 处理的事件（收到的短信、未接来电、应用通知和电池提醒）的副本转发到其他推送服务。常见的目标包括 Bark、Gotify、PushDeer、Lark 以及任意通用 Webhook。

每个目标都由一段 **HAR（HTTP Archive）请求**描述，应用会在每次事件发生时重放这段请求。由于请求是原样重放的，因此几乎任何基于 HTTP 的推送服务都能被支持，而无需为其单独编写集成代码。

## 打开抄送设置

1. 打开 Telegram SMS，点击溢出菜单（⋮）→ **抄送**。
2. 您会看到已配置的抄送服务列表（首次使用时为空）。

## 添加服务

1. 点击 **+** 悬浮按钮。
2. 将该服务的 HAR JSON 粘贴到编辑框中。编辑器会在您输入时校验 JSON，如果结构无效会显示错误。
3. 点击 **确定** 保存。

您也可以在溢出菜单中点击 **扫描** 选项，通过二维码导入服务。

> 为常见的服务商（Bark、Gotify、PushDeer、通用 cURL 请求……）获取结构正确的 HAR，最简单的方式是使用[配置生成器](https://config.telegram-sms.com/)。如果您想手动编写 HAR，请参阅开发者文档中的 [Carbon Copy Provider 参考](https://dev.telegram-sms.com/CarbonCopyProvider)。

### 占位符

在 HAR 正文中可以使用以下占位符，它们会在请求发送前针对每个事件被替换：

- `{{Title}}` — 消息标题（例如事件类型或短信发件人）。
- `{{Message}}` — 完整的消息正文。
- `{{Copy}}` — 适合复制的内容变体。

## 选择要抄送的事件

事件来源是一项**全局**抄送设置，对所有服务生效。打开溢出菜单并选择抄送配置项，然后切换您希望转发的来源：

- 收到的短信
- 未接来电
- 应用通知
- 电池提醒

只有在此处启用的来源才会被转发到您的抄送服务。

## 编辑、启用、停用或删除服务

- 点击列表中的某个服务可**编辑**其 HAR 或将其**删除**。
- 每个条目会显示其当前状态为 **(已启用)** 还是 **(已停用)**。

## 发送测试

使用溢出菜单中的 **发送测试** 可一次性向所有已配置的服务推送一条测试消息，以便在正式依赖它之前确认投递是否正常。如果尚未配置任何服务，应用会提示没有可测试的对象。

## 导入与加密

**接收配置** 选项允许您通过输入 ID 和密码从服务器拉取抄送配置。当配置为加密形式时，Telegram SMS 会使用您提供的密码进行解密（libsodium SecretBox），从而在传输过程中保护 HAR 中嵌入的凭据。

# カーボンコピー（Carbon Copy）

カーボンコピーは、Telegram SMS が処理するイベント（受信 SMS、不在着信、アプリ通知、バッテリー警告）のコピーを、Telegram への送信に加えて他のプッシュサービスにも転送します。よく使われる送信先には Bark、Gotify、PushDeer、Lark、および任意の汎用 Webhook があります。

各送信先は **HAR（HTTP Archive）リクエスト**で記述され、アプリはイベントごとにそのリクエストを再生（リプレイ）します。リクエストはそのまま再生されるため、専用の連携コードを書かなくても、ほぼすべての HTTP ベースのプッシュサービスに対応できます。

## カーボンコピー設定を開く

1. Telegram SMS を開き、オーバーフローメニュー（⋮）→ **カーボンコピー** をタップします。
2. 設定済みのカーボンコピーサービスの一覧が表示されます（初回は空です）。

## サービスを追加する

1. **+** フローティングアクションボタンをタップします。
2. サービスの HAR JSON をエディターに貼り付けます。エディターは入力中に JSON を検証し、構造が無効な場合はエラーを表示します。
3. **OK** をタップして保存します。

オーバーフローメニューの **スキャン** オプションから、QR コードでサービスをインポートすることもできます。

> 一般的なプロバイダー（Bark、Gotify、PushDeer、汎用 cURL リクエストなど）向けに正しい構造の HAR を入手する最も簡単な方法は、[設定ジェネレーター](https://config.telegram-sms.com/)を使うことです。HAR を手作業で組み立てたい場合は、開発者向けの [Carbon Copy Provider リファレンス](https://dev.telegram-sms.com/CarbonCopyProvider)を参照してください。

### プレースホルダー

HAR の本文では以下のプレースホルダーを使用できます。これらはリクエスト送信前にイベントごとに置換されます：

- `{{Title}}` — メッセージのタイトル（例：イベントの種類や SMS の送信元）。
- `{{Message}}` — メッセージ本文の全体。
- `{{Copy}}` — コピーしやすい形式の内容。

## コピーするイベントを選ぶ

イベントのソースは、すべてのサービスに適用される**グローバル**なカーボンコピー設定です。オーバーフローメニューを開いてカーボンコピー設定の項目を選び、転送したいソースを切り替えます：

- 受信 SMS
- 不在着信
- アプリ通知
- バッテリー警告

ここで有効にしたソースのみがカーボンコピーサービスに転送されます。

## サービスの編集・有効化・無効化・削除

- 一覧のサービスをタップすると、その HAR を**編集**したり、サービスを**削除**したりできます。
- 各項目には現在の状態として **(有効)** または **(無効)** が表示されます。

## テスト送信

オーバーフローメニューの **テスト送信** を使うと、設定済みのすべてのサービスに一度にテストメッセージを送信でき、本格的に利用する前に配信を確認できます。サービスがまだ設定されていない場合は、テスト対象がない旨が表示されます。

## インポートと暗号化

**設定の受信** オプションでは、ID とパスワードを入力してサーバーからカーボンコピー設定を取得できます。設定が暗号化されている場合、Telegram SMS は入力されたパスワードで復号します（libsodium SecretBox）。これにより、HAR に埋め込まれた認証情報が転送中も保護されます。

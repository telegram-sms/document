# 傳輸配置

傳輸配置可將一台設備上完整設定好的 Telegram SMS 複製到另一台設備，這樣您就不必手動重新輸入所有內容。透過應用程式右上角的溢出選單 (⋮) → **傳輸配置** 來打開它。

傳輸方式有兩種：一種是 **QR code** (離線，設備對設備)，另一種是透過中繼伺服器進行 **加密傳輸** (使用 ID 和密碼)。

## 包含的內容

傳輸的配置包含您完整的設定：

- 機器人令牌、API 位址和對話 ID
- 話題 ID 和可信的電話號碼
- 所有功能開關 (監控電池狀態、監控充電器狀態、獲取聊天命令、網絡錯誤回退到短信、驗證碼自動提取、來電通知、隱藏電話號碼)
- 您的抄送服務

## 方法一：QR code (離線，設備到設備)

在已經設定好的設備上，打開 **傳輸配置**。應用程式會顯示一個包含您配置資訊的 QR code。

在新設備上，使用主設定介面的 **掃描** (相機) 按鈕來掃描該 QR code。所有欄位將自動填入，然後點擊 **測試並保存** 完成設定。

> ⚠️ **請妥善保管此 QR code。** 它以明文形式包含了您的機器人令牌和所有配置，任何人掃描後都能控制您的機器人。應用程式會提示您：*"請妥善保管該 QR code，不要分享給他人！"*

## 方法二：加密傳輸 (ID + 密碼)

當兩台設備無法直接互相掃描時，您可以透過 Telegram SMS 的配置伺服器 (`api.telegram-sms.com`) 中繼配置。該配置會在您的設備上使用您設定的密碼進行加密。

### 發送配置

1. 在 **傳輸配置** 中，打開溢出選單 (⋮) → **發送配置**。
2. 輸入密碼 (至少 6 個字元)。該密碼用於加密您的配置；伺服器永遠不會獲取您的密碼。
3. 應用程式會上傳加密的配置並返回一個 **9 位字元的 ID**，該 ID 也會複製到您的剪貼簿。
4. 透過安全的管道將 ID 和密碼分享給目標設備。

### 接收配置

1. 在目標設備上，打開 **傳輸配置** → 溢出選單 (⋮) → **接收配置**。對於尚未設定的全新安裝，接收對話方塊會自動出現。
2. 輸入 **9 位字元的 ID** 和密碼。
3. 應用程式會下載並解密配置，自動填入所有欄位。點擊 **測試並保存** 完成設定。

> 配置在上傳之前，會在您的設備上使用 libsodium (SecretBox) 進行加密。因此，只有 **同時** 擁有 ID 和密碼的人才能讀取它。請選擇一個強密碼，並與 ID 分別發送。如果密碼錯誤，解密將會失敗，系統會提示您重試。

## 注意事項

- 加密傳輸需要網路能夠存取 `api.telegram-sms.com`。
- QR code 方式完全在離線狀態下運作。
- 這與 **抄送** 的「接收配置」功能使用的是相同的匯入機制。
